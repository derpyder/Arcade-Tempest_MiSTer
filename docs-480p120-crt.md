# 480p @ 120 Hz on a CRT PC monitor

**Short answer: yes, it works — and it's *less* demanding than the 720p120 HDMI mode
this core already ships** (the OSD's `120Hz (720p only)` option). It is set up entirely
in `MiSTer.ini` (no core rebuild), because for a framebuffer core the analog output
timing is generated by the MiSTer **scaler**, not by the game logic. The one hard
requirement is a **multisync** CRT PC monitor; a fixed-frequency 31 kHz VGA monitor
cannot do it.

---

## Why this is a config change, not a core change

This core renders vectors into a DDR3 framebuffer (`MISTER_FB`). The scaler reads that
buffer back and emits whatever output mode you set in `MiSTer.ini` — on its own clock,
decoupled from the core's `clk_108`/`CE_PIXEL`. So "480p120 on VGA" is a **scaler output
mode**, the same lever the OSD's `120Hz (720p only)` option pulls for HDMI, just pointed
at a different timing and at the analog DAC.

That OSD `120Hz` option is intentionally gated to 720p: it's a pixel-clock ceiling thing,
not an analog limit. 1080p120 (~300 MHz) is over the DE10-Nano's ~210 MHz ceiling; 720p120
(~150 MHz) fits. **480p120 needs only ~40–50 MHz**, with enormous headroom, and goes out
the analog VGA path rather than HDMI — so it's driven from the `.ini`.

## The numbers

| | Pixel clock | H-scan | Notes |
|---|---|---|---|
| 480p60 (standard) | 25.2 MHz | 31.5 kHz | any VGA monitor |
| **480p120 (CVT-RB)** | **~40–43 MHz** | **~60 kHz** | needs multisync CRT |
| **480p120 (full blanking)** | **~50 MHz** | **63 kHz** | needs multisync CRT |
| DE10-Nano ceiling | 210 MHz | — | huge margin |
| Analog DAC (ADV7125) | 330 MHz rated | — | not a limit |

## Setup (`/media/fat/MiSTer.ini`)

Use the analog VGA output (Analog I/O board) **or** Direct Video:

```ini
[MiSTer]
; --- custom 480p120 output ---
video_mode=640,480,120          ; MiSTer builds CVT-reduced-blanking timing
vsync_adjust=0                  ; emit the fixed mode (don't chase the core's 60 Hz)

; pick ONE output route:
vga_scaler=1                    ; route the scaler to the analog VGA port, OR:
;direct_video=1                 ; Direct Video (HDMI-to-VGA adapters / RGB)
;forced_scandoubler=1           ; (with direct_video)
```

If `640,480,120` is finicky on your monitor, use a full 9-value modeline
(`hact,hfp,hs,hbp,vact,vfp,vs,vbp,fpix_khz`) tuned to taste — see the MiSTer
[Video Modes doc](https://mister-devel.github.io/MkDocs_MiSTer/advanced/videomodes/).

## Monitor requirement (the real gate)

- **Needs a multisync CRT rated ≥ ~64 kHz horizontal.** Most 17″+ multisync PC CRTs
  (e.g. 30–85 kHz / 30–96 kHz tubes) qualify and routinely run 800×600@120 (~76 kHz).
- **A plain 31.5 kHz fixed-frequency VGA monitor will *not* sync** at 480p120 — that's
  the one thing that makes this harder than 240p120 (which keeps H-scan at ~31 kHz).

## Gotchas

- **Downscale softening.** The framebuffer is ~980×700; 640×480 output is a downscale,
  which filters vector lines. Pick a **sharp/nearest** scaler filter for crisp vectors.
- **60 fps content at 120 Hz repeats frames.** Tempest runs at 60 Hz, so a 120 Hz output
  shows each frame twice. On a CRT that can double-image; the upside is brightness and
  lower frame-pacing latency. This core's phosphor-persistence present-gate already
  smooths motion, so it's usually a non-issue — use black-frame insertion only if you see
  doubling. (A real Tempest tube relied on phosphor persistence anyway, which is exactly
  what the present-gate emulates.)
- **Test config-only first.** Because output timing lives in the scaler, this likely needs
  **zero** changes to the core. If a future build wants a dedicated analog-120 path it
  would be a small addition, but start with the `.ini` above.
